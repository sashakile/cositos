# Benchmark results

Same logical app per scenario, built two ways (A = naive peer-to-peer ipywidgets, B = MVU single-model). Each row is measured in an isolated subprocess. Generated by `run.py` — do not edit.

- `data_edges` — declared observe/link dependency edges (wiring complexity)
- `storm` — recomputes triggered by ONE user action
- `links_kept` — cross-part LinkModels that survive into the serialized document
